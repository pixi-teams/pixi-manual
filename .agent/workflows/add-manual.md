# マニュアル記事の追加

## 既存セクションに記事を追加する場合

1. `content/docs/` 配下の対象セクションディレクトリを確認する
   - `10-web-reservation/` — Web予約
   - `20-cast-management/` — キャスト管理
   - `30-admin/` — 管理機能

2. 連番ファイル名で Markdown ファイルを作成する
   - 命名規則: `{連番}-{スラッグ}.md`（例: `03-cancel-policy.md`）
   - 既存ファイルの最大連番 + 1 にする

3. 以下のテンプレートに従って内容を記述する

```markdown
# （機能名）

## できること
-

## 事前準備
-

## 手順
1.
2.
3.

## 確認ポイント
-

## よくあるミス
-
```

4. dev サーバーで表示を確認する
// turbo
```bash
pnpm dev
```

5. 対象ロールの目次ページと Book View に自動反映されていることを確認する
   - 目次: `/docs/{role}`
   - Book View: `/docs/{role}/book`

## 新しいセクション（章）を追加する場合

1. `content/docs/` に新しいセクションディレクトリを作成する
   - 命名規則: `{連番}-{スラッグ}/`（例: `40-analytics/`）

2. セクション内に Markdown ファイルを作成する（上記テンプレート使用）

3. `src/lib/mdx.ts` を編集して、新セクションをロールに紐付ける
   - `roleSections` に対象ロールへセクションディレクトリ名を追加
   - `sectionLabels` に表示ラベルを追加

4. dev サーバーで確認する
